# The Archiver — Agent Instructions

## Commit & Changelog Policy

Every code change MUST be committed before moving on. After each commit, update `CHANGELOG.md` following [Semantic Versioning](https://semver.org/):

- **MAJOR** — breaking changes to plugin API, database schema changes requiring manual migration, or config format changes
- **MINOR** — new features, new API endpoints, new plugin helpers
- **PATCH** — bug fixes, UI tweaks, dependency updates, docs

Add entries under an `## [Unreleased]` heading. When a release is cut, rename to `## [x.y.z] - YYYY-MM-DD`.

## Project Structure

- `src/app/` — Next.js App Router pages and API routes
- `src/components/` — React components (shadcn/ui in `ui/`, feature components in named folders)
- `src/db/` — Drizzle ORM schema, connection singleton, migration runner
- `src/lib/` — Server-side utilities (settings, notifications, kavita, events, ffmpeg)
- `src/plugins/` — Plugin registry, type definitions, and helper modules
- `src/workers/` — Download worker (SQLite polling, no Redis)
- `src/hooks/` — Client-side React hooks (SSE, data fetching)
- `plugins/` — User-installed plugins (each is a folder with `index.ts`)
- `drizzle/` — SQL migration files (generated by `drizzle-kit generate`)

## Key Architectural Decisions

- **No Redis** — The download worker polls SQLite directly. Do not introduce Redis or BullMQ.
- **Standalone output** — `next.config.ts` uses `output: "standalone"` for Docker. `better-sqlite3` is the only server external package.
- **Instrumentation hook** — `src/instrumentation.ts` runs DB migrations, seeds settings, and starts the worker at server boot. This is the app entry point for all server-side initialization.
- **Plugin system** — Plugins are loaded from both `plugins/` (user-installed) and the `installed_plugins` DB table. They export a `definePlugin()` definition with `urlPatterns` and a `download()` function.
- **SSE for real-time updates** — `src/lib/events.ts` broadcasts to connected clients. The worker emits events after queue state changes.
- **Settings system** — `src/lib/settings.ts` + `src/lib/settings-defs.ts`. Core settings are registered at boot from env vars with defaults. Plugin settings are registered dynamically.

## Database

SQLite via Drizzle ORM. Schema is in `src/db/schema.ts`. Tables: `settings`, `installed_plugins`, `download_queue`, `download_history`.

To add a schema change:
1. Modify `src/db/schema.ts`
2. Run `npm run db:generate` to create a migration
3. Migrations run automatically at server start via instrumentation hook

## UI / Styling

- **shadcn/ui** with the `nova` preset (Tailwind v4, radix base)
- **Fonts**: JetBrains Mono (headings/mono), DM Sans (body) — imported via `@fontsource-variable`
- **Theme**: "Vault" — dark industrial with amber/gold accents. Use semantic color tokens (`bg-primary`, `text-muted-foreground`), never raw color values
- **Icons**: `lucide-react` — use `size-*` for dimensions, `data-icon` on icons inside buttons

## Docker

Single-container deployment. The Dockerfile is a 3-stage build (deps → builder → runner). The runner includes `ffmpeg` via Alpine packages. Persistent volumes: `/data` (SQLite), `/downloads`, `/plugins`.

## CI/CD

GitHub Actions workflow at `.github/workflows/publish.yml` builds and pushes to `ghcr.io/pauljoda/the-archiver` on push to `main`. Multi-arch (amd64 + arm64) with GitHub Actions build cache.
